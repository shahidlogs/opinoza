Build a complete **Referral / Invite & Earn system** for my web app (Opinoza-style platform).

The system must be SIMPLE for users (no visible verification steps), but must include **strong hidden anti-fraud protections in the backend**.

---

# 🎯 CORE BUSINESS RULES

1. Every user has a unique referral code and invite link.

2. When a NEW user joins using a referral link AND signs in with Google:
   → The inviter earns **10 cents instantly**

3. When that invited user submits answers:
   → The inviter earns **0.5 cent per valid answer**

4. The inviter MUST NOT earn anything from:

   * invited user’s referrals
   * question creation
   * profile actions
   * any other activity

5. This is ONLY:

   * signup bonus
   * answer bonus

---

# 🔗 REFERRAL LINK SYSTEM

Each user gets:

* referral_code (unique)
* referral_link:
  /signup?ref=CODE

Example:
https://example.com/signup?ref=ABC123

---

# 📥 REFERRAL FLOW

## Step 1: User clicks invite link

* Capture referral_code from URL

* Store in:

  * session
  * cookie (7 days)

* Log click in database

---

## Step 2: Google Signup/Login

If user is NEW:

* Assign referred_by_user_id
* Create referral record
* Instantly give inviter **10 cents**

If user already exists:

* Ignore referral

---

## Step 3: Answer Reward

When invited user submits a valid answer:

* Check:

  * user has referrer
  * answer is valid (not deleted/spam)

* Give inviter:
  +0.5 cent per answer

---

# 🧠 IMPORTANT RULE

Each invited user is permanently linked to ONLY ONE referrer.

No switching allowed.

---

# 🛡️ HIDDEN ANTI-FRAUD SYSTEM (VERY IMPORTANT)

Do NOT ask users for verification.

Instead implement SILENT backend checks:

---

## 1. Self-referral detection

Block or flag if:

* same device fingerprint
* same browser
* same behavior pattern

---

## 2. IP monitoring

Track:

* signup IP
* referral click IP

Flag if:

* too many signups from same IP
* rapid account creation

---

## 3. Device fingerprinting

Store:

* user_agent
* device hash (basic fingerprint)

Flag if:

* same device creates multiple accounts

---

## 4. Google account uniqueness

Ensure:

* one Google account = one user

---

## 5. Suspicious pattern detection

Flag accounts if:

* multiple accounts created quickly
* no activity after signup
* repeated referral farming behavior

---

## 6. Reward control

Even though reward is "instant":

System must support:

* reversing rewards
* marking referral as:

  * pending
  * approved
  * flagged
  * rejected

---

## 7. Silent protection

DO NOT:

* show captcha
* show verification screens
* block normal users

Only flag in backend.
