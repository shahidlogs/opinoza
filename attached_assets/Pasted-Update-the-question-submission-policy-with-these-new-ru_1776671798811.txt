## RULE 1 — QUESTION SUBMISSION COST

Set the question submission cost to 25 cents.

* When a user submits a question, deduct 25 cents from their wallet
* Keep the current approval flow intact unless changes are needed for this pricing rule

---

## RULE 2 — REJECTION REFUND + PENALTY

If a submitted question is rejected by admin/moderation:

* refund 20 cents back to the user
* keep 5 cents as a penalty/fine

Store this clearly in the system so it is transparent.

Also:

* store the rejection reason
* store refunded amount
* store penalty amount

User notification text should clearly explain the rejection, for example:

"Your question was rejected because it appears to be spam, low-quality, unclear, duplicate, or against platform rules. 20 cents have been refunded to your wallet, and 5 cents were kept as a penalty."

Admin/moderator should be able to reject with a reason.

Transaction/history should remain clear, for example:

* Question submission: -25 cents
* Question rejection refund: +20 cents
* Penalty retained: 5 cents

If a question is approved:

* no refund is given
* the full 25 cents remains the submission cost

* previously submitted questions should remain as they are
